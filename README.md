# 📜 Governance Rules

All contributions must follow the **TEOS Forge Constitution**:

1. **Issues First**  
   - Every change begins with a documented Issue or Proposal.  
   - Use the provided templates in `.github/ISSUE_TEMPLATE`.

2. **Pull Requests (PRs)**  
   - PRs must reference an Issue.  
   - PRs are reviewed by Maintainers and Auditors.  
   - CI checks must pass before review.

3. **Audit Discipline**  
   - Audit findings override convenience and speed.  
   - Failed audits must be resolved before merge.  
   - Security disclosures follow [`SECURITY.md`](SECURITY.md).

4. **Role Clarity**  
   - **Contributor** → submits Issues/PRs.  
   - **Maintainer** → reviews PRs.  
   - **Auditor** → enforces compliance.  
   - **Steward (Elmahrosa International)** → governance escalation and treasury oversight.
