# TEOS Forge v0.1.0 — Governance Foundation

## Overview
This release establishes the **governance foundation** of TEOS Forge, transforming the repository into civic‑grade developer infrastructure.  
Phase 0 of the roadmap is now complete: governance before automation, audit before value, accountability before scale.

---

## Highlights
- Governance Constitution (`docs/governance.md`)
- Sustainability Model (`docs/monetization.md`)
- Badge definitions (`docs/badges.md`)
- Core module scaffolds:
  - People
  - Contribution
  - Accountability
  - Audit
  - Value Flow
- Issue & PR templates enforcing civic‑first rules
- CI pipeline with governance checks

---

## Governance Impact
- All contributions must follow **Issue → PR → CI → Review → Audit** flow
- Explicit role definitions: Contributor, Steward, Maintainer, Auditor
- Escalation and dispute resolution protocols in place
- Audit readiness checklist established
- Value‑flow rules defined (off‑chain, non‑executing)

---

## Audit & Compliance
- CI failures mapped to audit log triggers
- Security policy (`SECURITY.md`) enforced
- Contributor Covenant adopted (`CODE_OF_CONDUCT.md`)
- Institutional trust signals via badges row in `README.md`
